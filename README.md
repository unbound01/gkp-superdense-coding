# GKP Superdense Coding: Square vs. Hexagonal Lattice Comparison

Code accompanying the paper *"Superdense Coding with Gottesman-Kitaev-Preskill
Codes over Lossy Bosonic Channels: Square and Hexagonal Lattices Compared."*
[Link to paper — add on submission/arXiv upload]

Computes closed-form (square) and numerically-integrated (hexagonal) SDC
capacities for GKP-encoded Bell pairs transmitted over a lossy bosonic
channel, compares the two lattice geometries, and benchmarks against the
energy-matched Gaussian entanglement-assisted capacity $C_{\rm EA}$.

## Repo structure

```
src/
├── sweep.py          # core capacity formulas + full (s, eta) parameter sweep
├── figures.py         # regenerates Figs. 3, 5, 6 (from sweep output) and
│                       # Figs. 4, 7, 8 (fine-eta threshold curves, computed
│                       # directly from sweep.py's capacity functions)
├── fig2_voronoi.py    # Fig. 2 — Voronoi cell geometry, standalone, no
│                       # dependency on sweep.py's output
└── validation.py       # independent Monte Carlo + stage-by-stage circuit
                        # simulation cross-check
```

Fig. 1 (protocol circuit diagram) is a schematic, not a data plot, and is not
generated by code in this repo.

pip install -r requirements.txt

## Running

Run in this order from `src/`:

```bash
python3 sweep.py          # writes results/*.csv and results/all.npz
python3 figures.py        # writes figures/fig3-fig8 .pdf/.png (needs results/all.npz)
python3 fig2_voronoi.py   # writes figures/fig2_voronoi .pdf/.png (standalone)
python3 validation.py     # prints PASS/FAIL validation report to stdout
```

`results/` and `figures/` are generated, not committed — delete and
regenerate freely; if they disagree with what's committed, the code is the
source of truth, not any cached output.

## What `validation.py` actually checks

This is not a second run of the same formulas — it's an independently coded
cross-check, and the independence is the point:

- **Monte Carlo section**: draws Gaussian noise and classifies each sample by
  direct nearest-Euclidean-distance among the four true codeword coordinates
  $c_I, c_X, c_Y, c_Z$. It does **not** re-test the Voronoi half-plane
  inequalities that `sweep.py`'s analytic quadrature uses — so agreement
  between the two isn't circular. $N=8{,}000{,}000$ samples/point, 15
  $(s,\eta)$ points, seed 42.
- **Circuit section**: propagates two *separate* Gaussian noise injections
  (pre-amplifier vacuum noise and loss-channel vacuum noise, matching the
  physical stages individually) through the protocol and checks the output
  variance against the closed-form $\sigma_{\rm eff}^2 = 2\Delta^2+(1-\eta)$.
  $N=4{,}000{,}000$ samples/point, 7 $(s,\eta)$ points.

Current status: **all checks pass** (worst deviations: $|\Delta C_{\rm sq}|
=1.4\times10^{-3}$ bits, $|\Delta C_{\rm hex}|=3.85\times10^{-4}$ bits,
$|\Delta\sigma^2_{\rm eff}|=3.75\times10^{-4}$ — see script output for the
full per-point table).

## Known issue: environment used to generate reported numbers

The numbers currently in the manuscript were generated under NumPy 2.4.4 /
SciPy 1.17.1. If the manuscript's Appendix C states different pinned
versions (e.g. NumPy 1.26 / SciPy 1.12), that's a documentation mismatch —
resolve it before submission by either (a) re-running the full pipeline
under the versions stated in the paper and confirming the numbers don't
move, or (b) updating the paper's Appendix C to match the environment
actually used. Don't submit with the two disagreeing.

## Scope note

Earlier drafts of this work compared $C_{\rm sq}/C_{\rm hex}$ against the
unassisted Holevo capacity $C_{\rm hol}$ and claimed a "genuine superdense
coding advantage" over it. That comparison does not hold under the paper's
own energy-matching convention ($N_S=\Delta^{-2}$): $C_{\rm hol}$ exceeds the
2-bit GKP ceiling for all physically sensible squeezing in the swept range.
This repo's `sweep.py` still computes `C_hol` (for anyone who wants to
inspect this), but the current manuscript does not claim an advantage over
it — the paper's contribution is the hexagonal-vs-square lattice comparison
and the characterization against $C_{\rm EA}$, both fully verified above.
